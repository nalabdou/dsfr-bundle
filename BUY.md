## ✅ Éligibilité

> **Important** : Ce bundle est un wrapper du [Système de Design de l'État (DSFR)](https://github.com/GouvernementFR/dsfr).  
> L'utilisation du DSFR est **réservée aux sites et services numériques de l'État français**.

Pour pouvoir acquérir une licence Pro, votre projet doit obligatoirement :

- Être hébergé sur un domaine **`*.gouv.fr`** (ex. : `mon-service.gouv.fr`)
- Être autorisé à utiliser le DSFR conformément aux [conditions d'utilisation officielles](https://github.com/GouvernementFR/dsfr?tab=readme-ov-file#utilisation)

Toute demande provenant d'un domaine non éligible sera refusée.

> 🔒 **La licence est à vie**, liée à l'URL de production déclarée lors de l'achat. Elle ne nécessite aucun renouvellement.

## 📋 Informations requises

Merci d'inclure les éléments suivants dans votre message :

1. **Nom du projet / site web**
   Le nom commercial ou le nom du projet qui utilisera le bundle.

2. **URL du site en production**
   L'URL principale du site web (ex. : `https://mon-service.gouv.fr`).  
   ⚠️ Cette URL sera celle à laquelle la licence sera définitivement liée.

3. **URL(s) de test / staging**
   Les environnements non-production concernés (ex. : `https://staging.mon-service.gouv.fr`).

4. **Taille de l'équipe**
   Le nombre de développeurs qui auront accès au code source.

5. **Nature de l'équipe**
   Précisez si votre équipe est :
   - 🏢 Une équipe interne (agents publics dont le contrat est directement signé avec le service de l'État, sans prestataire intermédiaire)
   - 🏗️ Une ESN / SSII (prestataires externes intervenant pour le compte d'un service de l'État)
   - 👤 Un(e) ou plusieurs freelances indépendant(e)s
   - 🔀 Une combinaison des cas ci-dessus
